# 第34章: シナリオの「切り替え条件」— いつメインを捨てサブに移行するか

## Q1
**問題:** ケーススタディで、メインシナリオ通りの展開（If A）になった場合の対応として正しいものはどれか。
- A) 利確目標を延ばしてさらに利益を狙う
- B) メインシナリオの利確目標を手前に変更して確実に利益を確保する
- C) メインシナリオ通りにエントリーし、計画通りの損切り・利確を実行する
- D) サブシナリオも同時にエントリーする
**正解:** C
**解説:** If A（価格がZBに到達＋トリガーが出た）は最も理想的な展開。メインシナリオを計画通りに実行し、損切り・利確も事前に設計した通りに処理する。

## Q2
**問題:** ケーススタディで、サポートゾーンに到達したがトリガーが出ずに価格が反対方向に動いた（If B）場合、やってはいけないことはどれか。
- A) トリガーなしにゾーン到達だけでエントリーすること
- B) 次のゾーンまで待機すること
- C) アラートを再設定すること
- D) 見送り判断を下すこと
**正解:** A
**解説:** If Bの場合、「トリガーなしにゾーン到達だけでエントリー」してはならない。三位一体の確認（場所×足×出来高）を満たさないエントリーは確度が低く、損失の原因になる。

## Q3
**問題:** メインのサポートゾーンを下抜け→サブに移行（If C）したケーススタディにおいて、サブシナリオでトリガーが出なかった場合の正しい対応はどれか。
- A) サブシナリオの損切りラインを広げる
- B) ノートレードにする
- C) 別の時間足でトリガーを探す
- D) 逆方向にエントリーする
**正解:** B
**解説:** サブに移行した後もトリガーが出なければノートレードにする。無理にエントリーを探すと感情が介入し、判断の質が下がる。「やらない」という選択も立派なトレード判断。

## Q4
**問題:** 全シナリオが無効化（If D）された場合のケーススタディで、「撤退」が賢い判断とされる理由は何か。
- A) 損失が確定するので税金対策になるから
- B) 全シナリオが無効化されることは通常1年に1回しか起こらないから
- C) 無理に新しいシナリオを作ると焦りからバイアスのかかった分析になるから
- D) 撤退すれば必ず翌日に大きな利益のチャンスが来るから
**正解:** C
**解説:** 全シナリオが無効化されたときに無理に新しいシナリオを作ろうとすると、焦りから確証バイアス等が働き、質の低い分析になりやすい。その日は休んで翌日に冷静に再スタートするのが最善。

## Q5
**問題:** FOMCの結果が想定と大きく異なり、市場が急変したケーススタディでの対応として正しいものはどれか。
- A) 直後にメインシナリオを修正して即座にエントリーする
- B) 値動きが落ち着く（1〜2時間後）まで待ち、環境認識を再構築してから判断する
- C) FOMCの結果に関わらず、もとのシナリオをそのまま実行する
- D) 損切りラインを外してポジションを維持する
**正解:** B
**解説:** 重要指標でファンダメンタルズの前提が変わった場合、全シナリオを一時停止し、値動きが落ち着いてから環境認識を再構築する。慌てて飛び乗るのは最も危険な対応である。

## Q6
**問題:** シナリオ管理シートにおいて、メインシナリオとサブシナリオはどのタイミングで設計すべきか。
- A) メインシナリオが崩れてからサブを設計する
- B) エントリー後にサブを設計する
- C) 週に1回、週次レビューのときに設計する
- D) 朝のルーティンで同時に設計する
**正解:** D
**解説:** メインとサブは必ず「同時に」設計する。メインだけ先に作ってサブを後回しにすると、結局リアルタイムでサブを考えることになり感情が介入する。

## Q7
**問題:** シナリオ管理のルールのうち、「エントリー後にシナリオを変更しない」理由は何か。
- A) 変更すると税金の計算が複雑になるから
- B) 変更するとスプレッドが広がるから
- C) エントリー後の変更は100%感情によるものだから
- D) 変更するとブローカーのシステムエラーが起きるから
**正解:** C
**解説:** エントリー後に条件が変わったように見えても、それはポジションを持った心理状態による認知の歪みの可能性が高い。エントリー後のシナリオ変更は100%感情に由来すると考えてよい。

## Q8
**問題:** シナリオ管理シートで1日に管理すべきシナリオの上限はいくつか。
- A) 10通貨ペアまで
- B) 5通貨ペアまで
- C) 制限はない
- D) 最大2通貨ペアまで
**正解:** D
**解説:** 1日に管理するシナリオは最大2通貨ペアまで。3つ以上になると管理が行き届かなくなり、判断の質が下がる。少ないペアに集中することで精度の高いトレードが可能になる。

## Q9
**問題:** シナリオ管理シートの朝のルーティンに必要な追加時間はどの程度か。
- A) 約5分
- B) 約1時間
- C) 約30分
- D) 追加時間は不要
**正解:** A
**解説:** 朝のルーティンに5分追加するだけで、メイン＋サブの「2本立て」シナリオが完成する。この5分が日中の迷いを全て消し、判断を「計画の実行」に変えてくれる。

## Q10
**問題:** シナリオ管理シートを1週間続けるとどのような変化が起きると期待されるか。
- A) エントリー回数が3倍に増える
- B) チャートを開いた瞬間に「今日のIf-Aはここ、If-Cはここ」と頭に浮かぶようになる
- C) すべてのトレードで勝てるようになる
- D) 環境認識が不要になる
**正解:** B
**解説:** 1週間続けると、シナリオの分岐構造が身体に染み込み、チャートを開いた瞬間に条件分岐が自然と思い浮かぶようになる。これは準備が習慣化された証拠である。
