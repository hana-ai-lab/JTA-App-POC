CHAPTER 01
連勝バイアス
— 「俺は勝てる」が最も危険な瞬間
1-1 連勝バイアスとは
連勝バイアス（オーバーコンフィデンス）とは、連続して勝ったことで自分の能力を過大評価し、リスク管
理が甘くなる心理現象です。
3連勝した。5連勝した。「自分の分析は正しい。もっとロットを上げても大丈夫だ」——この思考の瞬間が
、口座を壊す起点になります。
WARNING 連勝バイアスが引き起こす5つの行動
1. ロットの増大 —
「勝てている間に稼ごう」とロットを上げる。1回の負けで連勝分の利益が吹き飛ぶ。
2. 損切りの先延ばし —
「今回も反転するだろう」と損切りを動かす。連勝が「自分は特別」という幻想を生む。
3. エントリー条件の緩和 —
「全ての条件が揃っていなくても、自分なら勝てる」とチェックリストを省略する。
4. 休むべき場面でトレードする —
「調子がいいから休むのがもったいない」とレンジ相場でも手を出す。
5. 分析の省略 — 環境認識やシナリオ構築を「面倒だから」と飛ばす。
1-2 なぜ連勝は危険なのか——数学的な真実
勝率60%のトレーダーが5連勝する確率は7.8%です。稀ではありますが、100回トレードすれば数回は起きま
す。
問題は、5連勝した本人がそれを「実力」と感じてしまうことです。7.8%の確率で起きる偶然を、「自分が
特別な何かを掴んだ」と誤認する。
連勝数 勝率60%での確率 正しい認識
3連勝 21.6% 5回に1回程度。十分に起きる。

CHECK 連勝時の3つのルール

CHAPTER 02
確証バイアス
— 「見たいものしか見えない」の罠
2-1 確証バイアスとは
確証バイアスとは、自分がすでに持っている仮説を支持する情報ばかりを集め、反する情報を無視してしま
う心理現象です。
「上昇トレンドだから買い」と決めた後、上昇を支持するチャートパターンだけが目に入り、下落の兆候が
見えなくなる。SNSで自分と同じ方向の意見だけをフォローし、反対意見を「わかっていない人の戯言」と

恐ろしいのは、確証バイアスは「分析力が上がるほど」強くなることです。分析力があるからこそ、自分の

CHAPTER 03
成功体験への執着
— 「前はこれで勝てた」の呪縛
3-1 成功体験はなぜ危険か
過去に大きな利益を出したトレードパターンに執着する。あの時と同じ形を探し、少しでも似ていれば飛び
つく。——これが成功体験への執着です。
「前回この形で大きく取れた。だから今回も同じだろう」——この推論には致命的な欠陥があります。相場
は二度と全く同じ状況にはならないのです。
3-2 成功体験への執着が引き起こす3つの問題
問題 詳細
① 過去の成功パターンに過度に最適化された判断をする。
パターンの 「あの時はZBでピンバーが出て200pips取れた」 →
過学習 似た形を見ると条件が揃っていなくてもエントリーしてしまう。
② 相場の 相場環境は常に変化している。ボラティリティ、
変化に 対 流動性、市場参加者の構成——全てが動的。
応できない 「前に通用した手法」が今も通用するとは限らない。
③ 新しい
「自分はもう十分に知っている」という錯覚。
学びの
成功体験が学習意欲を奪い、成長が止まる。
停止

習慣1：全てのトレードを「1回のサンプル」として扱う
過去の成功も失敗も、統計的なサンプルの1つにすぎない。「あの1回」に特別な意味を持たせない。100
回の結果で手法を評価する。
習慣2：月に1回「手法の棚卸し」をする
先月の勝ちパターンが今月も通用しているか確認する。通用していないなら、相場環境が変化した可能性
がある。手法を微調整する勇気を持つ。
習慣3：「初心者の目」で定期的にチャートを見る
過去の成功体験を一旦リセットし、まっさらな状態でチャートを分析する。先入観なしに「今、このチャ
ートは何を語っているか」を読む。
私自身、2020年のコロナ相場で大きく取れた経験があります。あの時の「暴落後の急反発」パターンが頭にこ
びりついて、2022年の下落相場でも「そろそろ反発するはず」と思い込んでいました。しかし2022年の下落は
性質が全く違った。コロナは「一時的なショック」でしたが、2022年は「構造的な利上げサイクル」だった。過去
の成功体験が判断を曇らせた典型的な失敗です。

CHAPTER 04
「勝ち始めた後」に現れる その他の心理的罠
4-1 プロスペクト理論の「第二段階」
基礎編で「利益は早く確定したくなり、損失は先延ばしにしたくなる」というプロスペクト理論を学びまし
た。これは初心者の罠です。
中級者になると、プロスペクト理論の「第二段階」が現れます。損切りはできるようになった。でも今度は
「利益を伸ばせない」という問題が出てくる。
POINT 中級者の利確問題
「含み益が出ている。ここで利確すれば勝ちトレード。でもシナリオ上は利確目標はもっと先……」
このとき、多くの中級者は利確目標より手前で決済してしまいます。「勝っているうちに利益を確定した
い」という心理。損切りができるようになっただけでは、トレードの問題は半分しか解決していないので
す。
対策はテーマ7で学んだ「分割利確」です。33%を先に確保すれば、残りを伸ばす心の余裕が生まれます。
心理的な問題を構造的に解決する——これがSTRADAのアプローチです。
4-2 比較の罠
SNSでは他のトレーダーの「勝ちトレード」が大量に流れてきます。「あの人は100万円稼いだのに、自分は
3万円しか稼げていない」——この比較が焦りを生み、無理なトレードに走らせます。
忘れないでください。SNSに流れてくるのは「勝ったトレード」だけです。負けたトレードは投稿されませ
ん。あなたが見ているのは、他人のハイライトリールであって、全体像ではない。
比較すべきは他人ではなく、「1ヶ月前の自分」です。1ヶ月前より環境認識が正確になったか。損切りが計
画通りにできたか。ルーティンを守れたか。自分の成長を測る基準は自分の中にあります。
4-3 「もう十分に稼いだ」の罠
月の前半で十分な利益が出ると、「もう今月は十分だ。残りは守りに入ろう」と思うかもしれません。一見
合理的ですが、これがバイアスを生むこともあります。
「守りに入る」が「条件が揃ってもエントリーしない」に変わってしまう。あるいは逆に、「ボーナストレ
ード」と称して普段やらないリスクの高いトレードに手を出す。

月の前半も後半も、判断基準は同じであるべきです。条件が揃えばエントリーする。揃わなければしない。
先月の損益も今月の損益も、次のトレードの判断には影響しない。
WARNING 「前回の結果」を次のトレードに持ち込まない
前回勝った→次も勝てる気がする（連勝バイアス）
前回負けた→次は取り返したい（リベンジトレード）
今月たくさん稼いだ→もう無理しなくていい（守りの過剰）
今月負けが多い→焦って条件を緩める（焦りのバイアス）
全て「前回の結果」が「次の判断」を歪めています。毎回のトレードは独立した事象です。1つ1つのトレ
ードを、その時点の環境認識とシナリオだけで判断してください。

CHAPTER 06
5つのケーススタディ
— 中級者が陥る心理的罠の実例
Case 1：5連勝後のロット倍増 — 連勝バイアスの典型
状況：5連勝して月間利益が過去最高。「この調子ならもっと稼げる」とロットを2倍にした。次のトレード
で大きく負け、5連勝分の利益の大半を失う。
分析：5連勝は勝率60%でも7.8%の確率で起きる。「実力」ではなく「確率」。ロットを変えなければ、1回
の負けは他の1回の負けと同じ程度の損失で済んだ。
教訓：連勝中こそロットを変えない。変えたくなったら「バイアスのサイン」と認識する。
Case 2：「買い」と決めてから買いの根拠だけ集めた — 確証バイアス
状況：ある通貨ペアで「日足上昇トレンドだから買い」と決めた。買いの根拠を5つ見つけた。しかし4時間
足の出来高が減少していること、VIXが上昇傾向であることを見落としていた。エントリー後にトレンドが
転換し損切り。
分析：「悪魔の代弁者」を実行していれば、出来高の減少とVIXの上昇を「反対の根拠」として検出できた
。
教訓：シナリオノートに「反対の根拠」欄を作る。メインの根拠と同数の反対根拠を探す習慣をつける。
Case 3：「前もこの形で勝った」で飛び乗った — 成功体験への執着
状況：過去にS級ゾーンでピンバーが出て大きく取れた経験がある。今回も似た形を見つけてエントリーし
たが、ゾーンの鮮度が3回目で、合流も1層しかなかった。結果は損切り。
分析：「形が似ている」だけで、条件が全く違った。鮮度3回目＝注文枯渇。合流1層＝根拠不足。チェック
リストを守っていれば見送りだったはず。
教訓：パターンの「形」ではなく「条件」で判断する。チェックリストは「成功体験のフィルター」として
機能する。

中級者のプロスペクト理論

状況：シナリオ上の利確目標はRR1:3。RR1:1.5の時点で「ここで確保しておこう」と全ポジション利確。そ

CHAPTER 06
5つのケーススタディ
— 指標前後の実戦判断
Case 1：CPI結果が予想を下回り、テクニカルと一致 —
「高確度のエントリー」
状況：ある通貨ペアでテクニカル上は下降トレンド継続中。CPI結果が予想を下回り（インフレ鈍化）、市
場は利上げペース鈍化を織り込んで通貨安方向に。テクニカルとファンダが一致。
対応：発表後1時間で値動きが落ち着いた後、4時間足のZSでトリガーを確認してショートエントリー。
Case 2：良い結果なのに売られた — 「織り込み済み」の典型
状況：雇用統計が予想を大幅に上回る好結果。初動で急上昇したが、30分後に急反転して大幅下落。
分析：発表前の1週間で市場はすでに好結果を織り込んで上昇していた。「噂で買って事実で売る」の典型
。初動の急上昇は反射反応、反転が本音。
対応：初動で飛び乗っていたら大損。2時間待って下落方向が確定した後に、テクニカルの根拠と合わせて
ショート検討。
Case 3：FOMC据え置きだが声明文がタカ派 —
「結果だけでは判断できない」
状況：市場予想通り金利据え置き。しかし声明文の文言が想定よりタカ派（利上げ示唆）。記者会見で追加
利上げの可能性が言及された。
対応：金利自体は変わっていないが、「将来の金利の方向」が変わった。環境認識のマクロ背景を「タカ派
寄り」にアップデートし、翌日以降の通貨高方向のバイアスに修正。
Case 4：指標前に含み益のポジションがあった — 「利確の判断」
状況：スイングトレードのロングポジションが含み益。翌日にCPI発表を控えている。
対応：3分割利確の第一利確分（33%）をCPI発表前に利確。残りはトレーリングストップを設定して保護。
発表後、結果がポジション方向と一致すれば継続。逆方向ならトレーリングが発動して自動利確。

CHAPTER 05
実戦シナリオ分析
— 5つのケーススタディ
Case 1：週足上昇×日足押し目×4H反転 — 教科書通りの押し目買い
5レイヤーすべてが「買い」を支持。高確度のエントリーポイント。4時間足のピンバー確定後にロングエン
トリー。損切りはフィボ61.8%の下。利確は直近高値の手前。
Case 2：週足レンジ×日足上昇 — 罠に注意の限定トレード
日足と4時間足だけ見れば「買い継続」だが、週足を見ると残り100pipsでレンジ上限に到達。利確目標をレ
ンジ上限手前に設定。ポジションサイズも50%に縮小。
Case 3：全時間足上昇だが出来高が減少 — 見えない危険信号
価格は上がっているのに出来高が直近3週間で明らかに減少。トレンドの「息切れ」のサイン。新規買いエ
ントリーは見送り。既存ロングはトレーリングストップを設定。
Case 4：テクニカルは買いだがCPI発表前 — 待つ勇気
テクニカル的には完璧な押し目買いのセットアップ。しかし翌日にCPI発表を控えている。発表前のエント
リーは見送り。発表後、セットアップが崩れていなければその時点でエントリー。
Case 5：トレンド転換の見極め — 最も判断が割れる局面
日足で直近の押し安値を割り込んだが、週足の200MAはまだ上向き。「わからない」が正解。週足の直近安
値まで待って反発を見るか、この通貨ペアから離れてもっと明確なペアを探す。
POINT 「わからない」を認める力
相場では「わからない」ことが頻繁に起きます。無理に答えを出そうとすると、バイアスのかかった判断
になります。わからないときは「わからない」と認めて、やらない。これが勝ち組と負け組を分ける最大
の分岐点です。

「この上昇トレンドは加速中なのか、減速中なのか」「4時間足のこの下落は、日足の修正波なのか、転換

ば、このレポートは成功です。

1-1 シナリオは「予言」ではなく「準備」

2-1 メインシナリオの定義

の条件が揃っているシナリオ。
2-2 サブシナリオの定義
サブシナリオとは、「メインシナリオが崩れた場合に発動する」代替シナリオです。メインが「買い」なら
、サブは「見送り」or「売り」になることが多い。
重要なのは、サブシナリオは「メインが崩れてから考える」のではなく「メインと同時に設計しておく」こ
とです。

シナリオの切り替えで最も重要なのは、「いつ切り替えるか」を事前に決めておくことです。リアルタイム

CHAPTER 04
「想定外」をなくす思考法
— If-Then-Elseの構造化
4-1 プログラマー的思考をトレードに応用する
プログラミングの世界では、「If（もし〇〇なら）→ Then（こうする）→ Else（そうでなければこうする
）」という条件分岐が基本です。トレードのシナリオも同じ構造で設計できます。
この構造の最大のメリットは、「想定外」がなくなることです。全ての分岐を事前に定義しておけば、どん
な値動きが来ても「これはIf-Aのケースだな」「これはElseのケースだな」と冷静に対応できます。
4-2 3段階のIf-Then-Else
CHECK シナリオの条件分岐テンプレート
If A：価格がZBに到達＋トリガーが出た
→ Then：メインシナリオ実行（エントリー）
If B：価格がZBに到達したが、トリガーが出ない
→ Then：待つ。2本のローソク足以内にトリガーが出なければ見送り
If C：価格がZBを終値で下抜けた
→ Then：メインを破棄。サブシナリオ（より深いZB）に切り替え
If D：サブシナリオのZBも下抜けた
→ Then：全シナリオ無効化。この通貨ペアは今日はやらない
Else：上記のいずれにも該当しない状態
→ Then：まだ何も起きていない。アラートを設定して待つ
4-3 分岐図をノートに書く
このIf-Then-Elseを、実際にノートに分岐図として書いてください。ツリー構造で描くと、全体像が一目で
把握できます。
私は毎朝、その日のシナリオをノートに分岐図で書いています。所要時間は5分程度。この5分が、日中の判断
を全て「計画の実行」に変えてくれます。分岐図がない日のトレードは、地図なしで知らない街を歩くようなもの

CHAPTER 05
5つのケーススタディ
— 複数シナリオの実戦運用
Case 1：メインシナリオ通りに進行 — 「計画通り」
状況：ある通貨ペア。メインシナリオ＝日足ZBでの押し目買い。価格がZBに到達し、4H足で包み足が出現。
出来高も伴っている。
対応：If Aに該当。メインシナリオを実行。エントリー→利確。最も理想的な展開。
Case 2：ZBに到達したがトリガーが出ない — 「待つ→見送り」
状況：ある通貨ペア。メインシナリオ＝4H足ZBでの押し目買い。価格がZBに到達したが、明確なトリガーが
出ないまま2本のローソク足が経過。
対応：If Bに該当。見送り。「トリガーなしにゾーン到達だけでエントリー」はしない。
Case 3：メインのZBを下抜け→サブに移行 — 「計画的な切り替え」
状況：ある通貨ペア。メインのZB（ある水準A〜B）を4H足の終値で下抜け。事前に設定していたサブシナリ
オのZB（ある水準C〜D）に価格が接近中。
対応：If Cに該当。メインを破棄し、サブシナリオに切り替え。サブのZBでトリガーを待つ。
Case 4：全シナリオが無効化 — 「撤退する勇気」
状況：ある通貨ペア。メインもサブも下抜けた。日足の直近安値も割り込んだ。上昇トレンドの前提自体が
崩壊。
対応：If Dに該当。全シナリオ無効化。この通貨ペアから離れて、別のペアで環境認識からやり直す。
POINT 「撤退」は最も賢い判断
全シナリオが無効化されたとき、「何もしない」が正解です。無理に新しいシナリオを作ろうとすると、
焦りからバイアスのかかった分析になります。その日は休んで、翌日に冷静な状態で再スタートしてくだ
さい。

おわりに
— シナリオは「予言」ではなく「準備」 —
このレポートの核心は、シナリオに対する認識の転換です。
シナリオは「当てる」ものではなく「備える」ものです。
メインシナリオが当たれば利益。外れたらサブシナリオに切り替え。サブも崩れたら撤退。この全てが「計
画通り」です。計画通りに撤退できたことは、感情で損切りを先延ばしにするよりはるかに価値があります
。
相場の未来は誰にもわかりません。でも「もしこうなったら、こうする」を全て準備しておくことはできま
す。その準備の量が、あなたのトレードの質を決めます。
わかる範囲でわかることを。できる範囲でできることを。自分の頭で考えて。
