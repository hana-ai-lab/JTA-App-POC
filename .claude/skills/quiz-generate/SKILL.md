# Quiz Generate

JTAナレッジベースから高品質な4択クイズを生成するスキル。

## 前提

- ナレッジファイル:
  - `docs/lite-kohaku_knowledge_v3.md` — 入門講座（コハクくん）、Week 1-12
  - `docs/basic-pro-kohana_knowledge_v3.md` — Basic講座（コハナちゃん）、Week 1-12
- 講義資料（内容確認・レビュー用）:
  - `docs/講義資料/01_入門講座_駆け込み寺.md` — 入門講座 全12週
  - `docs/講義資料/` 配下の該当ファイル
- 出力先: `docs/quizzes/{course}/week-{NN}.md`
- 1ファイル = 1 Week = 20問（ユーザーが別の数を指定した場合はそちらに従う）

## 手順

### 2. ナレッジの読み込み

該当Weekのセクションをナレッジファイルから読み取る。

- kohaku: `### Week N` から次の `###` まで
- kohana: `#### タイトル（Week N）` から次の `####` まで

必要に応じて講義資料も参照し、内容を十分に理解してからクイズを作成する。

### 3. クイズ生成

以下のルールに従って問題を生成する（デフォルト20問、ユーザー指定があればそちらに従う）。

#### 文面のルール（最重要）

- **資料への言援を入れない**: 問題文・解説に「講義では」「ナレッジでは」「Akiraは〜と述べている」等の、資料が存在することを前提とした表現を入れない。クイズは単体で成立する一般的な知識問題として書く。
- **不正解は全てあり得る選択肢にする**: どれを選んでも「それもあり得る」と思えるレベルのもっともらしさを持たせる。明らかにあり得ない・おかしい選択肢は避ける。よくある誤解や、一見正しそうに見える内容を不正解に使う。

#### 質の基準

- **網羅性**: そのWeekの主要ポイントをまんべんなくカバーする
- **深さのバリエーション**: 単純な用語確認（5問程度）、概念理解（8問程度）、応用・判断（7問程度）を混ぜる
- **正解の位置**: A〜Dに均等に分散させる（各5問ずつが目安）
- **解説は簡潔に**: 1〜2文でWhyを説明する。長く書きすぎない。資料への言援はしない。
- **実践的**: トレーダーが実際に直面する場面を想定した問題を含める

#### 避けるべきパターン

- 「次のうち正しいものはどれか」のような曖昧な問い方
- 「すべて」「絶対に」などの極端な表現（正解にも不正解にも）
- ナレッジに書いていない内容からの出題
- 数値の丸暗記だけを問う問題（理解を問う形にする）
- 「〜について説明されているのは」「〜で挙げられている」等の資料前提の問い方

### 4. 出力

`docs/quizzes/{course}/week-{NN}.md` に以下のフォーマットで書き出す。

```markdown
---
course: kohaku
week: 1
title: "なぜ負けるのか — 9割が退場する本当の理由"
chapter_id: null
available_for_light: false
---

# Week 1: なぜ負けるのか — 9割が退場する本当の理由

## Q1
**問題:** 個人投資家の退場パターンには「一発退場型」「ジリ貧型」「休眠型」の3つがあります。この中で最も多い退場パターンはどれですか？
- A) 一発退場型 — レバレッジをかけすぎて一度の暴落で資金の大半を失うパターン
- B) ジリ貧型 — 毎回少しずつ負けて気づいたら資金が半分以下になるパターン
- C) 休眠型 — 大きな負けを経験して精神的に相場に向き合えなくなるパターン
- D) 情報不足型 — 投資知識が不足し、根拠のないトレードを繰り返して退場するパターン
**正解:** B
**解説:** ジリ貧型が最も多い退場パターンです。勝ちトレードの利益より負けトレードの損失がいつも大きく、じわじわと口座が減っていき気づきにくいため危険です。

## Q2
**問題:** ...
- A) ...
- B) ...
- C) ...
- D) ...
**正解:** C
**解説:** ...

(Q3〜Q20まで続く)
```

**フロントマターの説明:**
- `course`: kohaku または kohana
- `week`: Week番号（数値）
- `title`: Weekのタイトル
- `chapter_id`: アプリDBのchapter_id（ユーザーに確認、または既存の最大ID+1から採番。不明なら `null`）
- `available_for_light`: Liteプランで表示するか（デフォルト false）

### 5. 確認

生成後、以下をユーザーに報告する:
- 出力ファイルパス
- 問題数
- 正解の分布（A/B/C/D 各何問か）
- 難易度の分布（用語確認/概念理解/応用・判断）
